План работы л/р №3 и №4

1. Доработать сформированные модели (AR).
проверить и добавить правила проверки данных:
- email должен иметь корректный формат
- цена должна обязательно задаваться в формате с точкой (например: 129.33)
- имя пользователя (login/email) должно быть уникальным
- пароль должен быть не менее 6 символов и не больше 10
- проверить пароль на надежность (буквы, цифры, регистр)
обеспечить хранение шифрованного пароля в базе данных

2. Административная часть сайта.
сгенерировать модуль “admin”
добавить фильтры доступа только для пользователей с ролью “admin” (если нету роли – добавить)
сформировать базовый Controller для админчасти (с фильтрами доступа)
перенести CRUD-ы в модуль или создать снова внутри модуля
сформировать меню административной части (добавить в него все CRUD-ы)
доработать созданные CRUD-ы для смысловых сущностей (Категории, Продукты, Пользователи и т.д.)
- убрать ненужные поля из таблиц и из форм
- добавить загрузку и сохранение картинок (потребуется доработка модели) через форму редактирования
- обеспечить возможность выбирать значения из словаря (например Категорию при редактировании Продукта из списка из БД)

3. Представить проект в GitHub.
файлы проекта
схема базы данных в файле app\db.sql
